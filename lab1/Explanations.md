**Мост (bridge) и коммутатор (switch)**  

__Мост (bridge)__  
1. Уровень работы: Мост работает на уровне канала данных (Data Link Layer) в модели OSI (чаще всего на уровне 2, но могут быть и на уровне 3). Он анализирует кадры данных, которые проходят через него, и определяет, следует ли передавать их на другой порт или отбросить.  
2. Решение проблемы коллизий: Мост используется для разделения сегментов сети и предотвращения коллизий. В прошлом, когда сети Ethernet были разделены на сегменты с коллизиями.  
3. Принимает решение на основе MAC-адресов: Мост работает на основе MAC-адресов устройств. Он изучает MAC-адреса, связанные с каждым портом.
   
```sudo apt-get install bridge-utils``` - Установить bridge-utils, который содержит утилиту brctl  
```sudo ip link add name <name> type bridge``` -  Создать мост  
```sudo ip link set dev <name> up```  
```sudo brctl show <name>``` - Посмотреть информацию о мосте  
```sudo brctl delbr <name>``` - Удалить  
```
topology:

      kind: bridge
  
  links:
    - endpoints: ["R01:eth1", "br-clab:eth1"]
    - endpoints: ["Linux1:eth1", "br-clab:eth2"]
```

__Коммутатор (switch)__  
1. Уровень работы: Коммутатор также работает на уровне канала данных (Data Link Layer) в модели OSI. Коммутатор более интеллектуальное устройство, чем мост.  
2. Уменьшает трафик: Коммутатор уменьшает трафик в сети, пересылая данные только на порт, где находится устройство-получатель.  
3. Принимает решение на основе MAC-адресов: Как и мост, коммутатор также использует MAC-адреса для принятия решения о пересылке данных, но он делает это на более высоком уровне.
